SQL Style

SQL is easy to learn but so broad that mastery takes extensive time. The features and techniques I'm sharing here are those that make getting insights cheap, easy and maintainable. After a few 100,000 lines of SQL, I have some very strong opinions on the language. These insights come from not just volume but having worked with a diverse array writing SQL on everything from large scale 3NF databases and optimized analytics tables to unstructured jsonb blobs (*shivers at the thought).

Dealing With Time-Series Events

There are 3 tools I love to use time and time again when dealing with time. These are (1) row_number() paired with partition by X order by Y, (2) lag(M, #) paired with partition by X order by Y, and extensive use of case statements so I can tell my actions apart if multiple nuances exist. Using these things you can easily create 


                    SELECT 
                        a,
                        b,
                        c
                    FROM 
                        abc

We complete these tasks in many manual ways through code and processes, however the goal is to systemetize this so we can leverage far more powerful tools than are possible to be built on top of every manual solution. Simply put, the goal is to achieve economies of scale Users new to the idea of economies of scale can see Wikipedia for more information. with regards to information we can inject given an ever increasing amount of labor, with the key value proposition being that this type of monitoring will dramatically improve a model builders intuition and potentially even reduce the anxeity of deploying more models. While I deeply dislike Facebook, their goal to create the Newspaper for you is very much inline with what I am trying to do here. Instead of a Newspaper, I want this to be your view of the world you've automated, which are bound to require tune-ups. Better to know whats going on than wait until things break and you lose some value which you had initially gained by building a prediction in the first place.
